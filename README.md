# Amazon Platform Case Study & Database Design

## Platform Understanding
- Studied Amazon’s core features: product search, ordering, payments, delivery tracking, and reviews  
- Analyzed how users interact with the platform via web and mobile apps

## Problem-Solving Analysis
- Solves issues like limited product access, time-consuming shopping, and price comparison  
- Features such as fast delivery, easy returns, and reviews improve user trust and convenience

## Case Study Development
- User journey: Search → Compare → Add to Cart → Payment → Delivery → Review  
- Personalized recommendations enhance user experience

## Schema Design
- Designed database with entities: User, Product, Order, Payment, Seller, Review, Shipment  
- Defined clear relationships using primary and foreign keys

## Rationale & Strategy
- Normalized schema to reduce redundancy  
- Design supports scalability, performance, and data integrity

## ER Diagram
- Created ER diagram showing relationships between users, orders, and products

## Presentation & Creativity
- Clear documentation and structured solution  
- Simple, professional, and real-world focused design
